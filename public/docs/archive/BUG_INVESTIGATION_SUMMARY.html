Bug Investigation Summary - KAN-48 Testing

Overview

This document summarizes the investigation and testing work completed for two critical bugs discovered during KAN-48 implementation testing.

Branch Information

  • Branch: KAN-48-AUTOMATION-TESTS
  • Base: main
  • Purpose: Bug reproduction tests and fixes

Bugs Identified

Bug 1: Interns Cannot View Their Application Requests ❌

Severity: High
Impact: Interns have no visibility into their submitted applications

Problem Description

When an intern submits an application to a project via the “I’m Interested” button: 1. Application is successfully submitted to /api/join-requests 2. Success notification appears 3. BUG: Intern has no way to view their submitted application status

Root Cause Analysis

  • RequestsTab.tsx only fetches /api/incoming-requests
  • This endpoint shows requests TO projects the user manages (for leaders)
  • No endpoint exists for /api/my-applications or /api/outgoing-requests
  • Interns cannot track their application status (pending, approved, declined)

User Flow (Current - Broken)

Intern → Opportunities Tab → Click "I'm Interested" → Submit Application
                                                              ↓
                                                    Success Notification
                                                              ↓
                                                         ❌ NOT FOUND
                                                    (No view to see application)

Test Coverage

Created RequestsTab.test.tsx with: - ✅ Test: Intern can render RequestsTab - ❌ Test: Should show “My Applications” tab (FAILS - bug exists) - ❌ Test: Should fetch user’s own applications (FAILS - endpoint missing) - ✅ Test: Leaders can fetch incoming requests (works correctly) - ✅ Test: Documents that interns have no tabs (current broken state)

Bug 2: Leaders Cannot Create Projects from Your Projects Tab ❌

Severity: Medium
Impact: Leaders must navigate to different tab to create projects

Problem Description

When a leader navigates to “Your Projects” tab: 1. Can view managed and contributing projects 2. BUG: No “Create Project” button available 3. Must go to “Opportunities” tab to create projects

Root Cause Analysis

  • YourProjectsTab.tsx doesn’t include create button
  • CreateProjectModal component exists and works
  • Button is available in AmbitionsTab but not YourProjectsTab
  • Inconsistent UX - leaders expect to create from “Your Projects”

User Flow (Current - Broken)

Leader → Your Projects Tab → View Projects
                                    ↓
                            ❌ NO CREATE BUTTON
                                    ↓
                    Must navigate to Opportunities Tab
                                    ↓
                            Click "Create Project"

Test Coverage

Created YourProjectsTab.test.tsx with: - ✅ Test: YourProjectsTab renders for leaders - ✅ Test: YourProjectsTab renders for managers - ❌ Test: Should have Create Project button for leaders (FAILS - bug exists) - ❌ Test: Should have Create Project button for managers (FAILS - bug exists) - ✅ Test: Documents that AmbitionsTab has the button (for comparison)

Proposed Fixes

Fix for Bug 1: Add “My Applications” View for Interns

Backend Changes Needed

  1. Create new API endpoint: GET /api/my-applications
    • Fetch applications where user_id = current_user
    • Include project details, status, dates
    • Return application history

Frontend Changes Needed

  1. Modify RequestsTab.tsx:
    • Add role detection for interns
    • Add “My Applications” tab for interns
    • Fetch from /api/my-applications endpoint
    • Display application cards with:
      • Project name
      • Application status (pending, approved, declined)
      • Date submitted
      • Any feedback from PM
    • Allow withdrawal of pending applications

Implementation Steps

// In RequestsTab.tsx
const [myApplications, setMyApplications] = useState<any[]>([]);

const fetchMyApplications = async () => {
  const response = await fetch('/api/my-applications', { credentials: 'include' });
  const data = await response.json();
  setMyApplications(data);
};

// Add tab for interns
{!isLeaderOrManager && (
  <Tab>My Applications ({myApplications.length})</Tab>
)}

Fix for Bug 2: Add Create Project Button to Your Projects Tab

Changes Needed

    • Import CreateProjectModal
    • Add state for modal: const [showCreateModal, setShowCreateModal] = useState(false)
    • Add “Create Project” button in header (for leaders/managers only)

Implementation Steps

// In YourProjectsTab.tsx header section
