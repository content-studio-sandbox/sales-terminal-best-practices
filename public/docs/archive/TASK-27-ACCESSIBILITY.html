TASK-27: Accessibility Pass

Overview

Implement comprehensive accessibility improvements including keyboard navigation, focus management, ARIA labels, and screen reader support.

Accessibility Standards

  • WCAG 2.1 Level AA compliance target
  • Keyboard navigation for all interactive elements
  • Screen reader compatibility
  • Focus management for modals and dynamic content
  • Color contrast meeting 4.5:1 minimum ratio

Key Areas for Improvement

1. Keyboard Navigation

Requirements: - All interactive elements accessible via Tab/Shift+Tab - Enter/Space to activate buttons and links - Escape to close modals and dropdowns - Arrow keys for list navigation - Focus visible indicators

Components to Update: - Modal dialogs (all modals) - Dropdown menus (filters, selects) - Data tables (sortable columns) - Card grids (project cards, career path cards) - Tab navigation (main tabs)

2. Focus Management

Requirements: - Focus trap in modals (prevent tabbing outside) - Return focus to trigger element on modal close - Skip to main content link - Logical focus order - Visible focus indicators

Implementation:

// Focus trap hook for modals
const useFocusTrap = (isOpen: boolean, modalRef: RefObject<HTMLElement>) => {
  useEffect(() => {
    if (!isOpen) return;
    
    const modal = modalRef.current;
    if (!modal) return;
    
    const focusableElements = modal.querySelectorAll(
      'button, [href], input, select, textarea, [tabindex]:not([tabindex="-1"])'
    );
    
    const firstElement = focusableElements[0] as HTMLElement;
    const lastElement = focusableElements[focusableElements.length - 1] as HTMLElement;
    
    const handleTab = (e: KeyboardEvent) => {
      if (e.key !== 'Tab') return;
      
      if (e.shiftKey) {
        if (document.activeElement === firstElement) {
          e.preventDefault();
          lastElement.focus();
        }
      } else {
        if (document.activeElement === lastElement) {
          e.preventDefault();
          firstElement.focus();
        }
      }
    };
    
    modal.addEventListener('keydown', handleTab);
    firstElement?.focus();
    
    return () => modal.removeEventListener('keydown', handleTab);
  }, [isOpen, modalRef]);
};

3. ARIA Labels and Roles

Requirements: - Descriptive labels for all form inputs - ARIA labels for icon-only buttons - Role attributes for custom components - Live regions for dynamic content - Landmark roles for page sections

Components Needing ARIA: - Icon buttons (edit, delete, close) - Search inputs - Filter dropdowns - Status indicators - Loading states - Error messages

Example Implementation:

// Icon button with ARIA label
<Button
  kind="ghost"
  size="sm"
  renderIcon={Edit}
  iconDescription="Edit project"
  aria-label="Edit project"
  onClick={handleEdit}
/>

// Live region for status updates
<div role="status" aria-live="polite" aria-atomic="true">
  {statusMessage}
</div>

// Search with clear label
<Search
  id="project-search"
  labelText="Search projects"
  placeholder="Search by title or description..."
  aria-label="Search projects by title or description"
/>

4. Screen Reader Support

Requirements: - Meaningful page titles - Descriptive link text (no “click here”) - Alt text for images - Table headers properly associated - Form validation messages announced - Loading states announced

Implementation Examples:

// Announce loading state
<div role="status" aria-live="polite">
  {isLoading ? "Loading projects..." : `${projects.length} projects loaded`}
</div>

// Table with proper headers
<Table>
  <TableHead>
    <TableRow>
      <TableHeader id="project-name">Project Name</TableHeader>
      <TableHeader id="status">Status</TableHeader>
    </TableRow>
  </TableHead>
  <TableBody>
    <TableRow>
      <TableCell headers="project-name">Project Alpha</TableCell>
      <TableCell headers="status">Active</TableCell>
    </TableRow>
  </TableBody>
</Table>

// Form validation
<TextInput
  id="project-title"
  labelText="Project Title"
  invalid={!!errors.title}
  invalidText={errors.title}
  aria-describedby={errors.title ? "title-error" : undefined}
/>
{errors.title && (
  <div id="title-error" role="alert">
    {errors.title}
  </div>
)}

5. Color and Contrast

Requirements: - Text contrast ratio ≥ 4.5:1 (normal text) - Text contrast ratio ≥ 3:1 (large text) - UI component contrast ≥ 3:1 - Don’t rely on color alone for information - Support high contrast mode

Areas to Check: - Button text on colored backgrounds - Link colors - Status indicators - Disabled states - Error messages
