Executive Dashboard Setup Guide

This guide will help you set up the Executive Dashboard with performance tracking and project reviews.

Prerequisites

  • Supabase project with authentication enabled
  • User with leader or manager access_role
  • Tables: users, projects, user_performance, project_reviews

Step-by-Step Setup

Step 1: Apply RLS Policies

The 406 error you’re seeing is because Row Level Security (RLS) is not configured for the new tables.

Run this in Supabase SQL Editor:

-- Copy and paste the contents of APPLY_RLS_POLICIES.sql

Or navigate to: Supabase Dashboard → SQL Editor → New Query and paste the contents of APPLY_RLS_POLICIES.sql

This will: - Enable RLS on user_performance and project_reviews tables - Add policies allowing authenticated users to read all records - Add policies allowing leaders/managers to manage performance data - Add policies allowing project managers to review their projects

Step 2: Populate Default Performance Data

After applying RLS policies, populate default performance data for existing users.

Run this in Supabase SQL Editor:

-- Copy and paste the contents of APPLY_DEFAULT_PERFORMANCE_DATA.sql

Or navigate to: Supabase Dashboard → SQL Editor → New Query and paste the contents of APPLY_DEFAULT_PERFORMANCE_DATA.sql

This will: - Create default performance records for existing users with access_role IN (‘user’, ‘intern’, ‘contributor’) - Excludes leaders and managers (they rate others, not themselves) - Set default rating to meets_expectations - Set default pipeline stage to active - Skip users who already have performance records

Step 3: Verify the Setup

After running both SQL files, verify the setup:

  1. Check RLS Policies:

    SELECT schemaname, tablename, policyname, permissive, roles, cmd
FROM pg_policies
WHERE tablename IN ('user_performance', 'project_reviews')
ORDER BY tablename, policyname;

  2. Check Performance Data:

    SELECT 
    u.display_name,
    u.email,
    up.performance_rating,
    up.pipeline_stage
FROM users u
LEFT JOIN user_performance up ON u.id = up.user_id
ORDER BY u.display_name;

  3. Refresh your application and navigate to the Leadership Dashboard

Features Now Available

Leadership Dashboard

  1. Performance Rating Column
    • Dropdown with 5-star ratings
    • Maps to TEXT values: high_performer, meets_expectations, needs_improvement
    • Leaders/managers can rate any user
    • Users can rate themselves
    • Note: Leaders and managers show “N/A” (they are not rated)
  2. Pipeline Stage Column
    • Dropdown with talent pipeline stages
    • Options: Active, High Performer, Conversion Candidate, Offer Extended, Offer Accepted
    • Tracks progression through hiring/conversion pipeline
    • Note: Leaders and managers show “N/A” (they manage the pipeline, not in it)

Executive Dashboard

  1. Real Metrics
    • Program ROI calculated from actual project reviews
    • Business Value summed from project.business_value
    • Talent Pipeline counts from user_performance.pipeline_stage
    • Strategic Alignment based on quality and business impact ratings
  2. Self-Service Configuration
    • Configure targets, goals, and formulas
    • Manage strategic initiatives
    • Set diversity targets

Project Reviews

  1. Review Completed Projects
    • Quality rating (1-5 stars)
    • Business impact rating (1-5 stars)
    • Innovation score (1-5 stars)
    • Actual ROI tracking
    • Lessons learned

Troubleshooting

Still seeing 406 errors?

  1. Verify RLS is enabled:

    SELECT tablename, rowsecurity 
FROM pg_tables 
WHERE tablename IN ('user_performance', 'project_reviews');

    Both should show rowsecurity = true

  2. Verify policies exist:

    SELECT COUNT(*) as policy_count, tablename
FROM pg_policies
WHERE tablename IN ('user_performance', 'project_reviews')
GROUP BY tablename;

    Should show multiple policies for each table

  3. Check your user’s access_role:

    SELECT id, email, access_role FROM users WHERE id = auth.uid();

    Should return your current user with a valid access_role

Dropdowns still empty?

  1. Verify performance data exists:

    SELECT COUNT(*) FROM user_performance;

    Should return > 0

  2. Check if your user has permission to read:

    SELECT * FROM user_performance LIMIT 5;

    Should return rows without errors

Database Schema Reference

user_performance table

- id: UUID (primary key)
- user_id: UUID (foreign key to users.id)
- performance_rating: TEXT ('high_performer', 'meets_expectations', 'needs_improvement')
- pipeline_stage: TEXT ('active', 'high_performer', 'conversion_candidate', 'offer_extended', 'offer_accepted')
- rated_at: TIMESTAMP

Important: Only users with access_role IN (‘user’, ‘intern’, ‘contributor’) should have performance records. Leaders and managers are excluded from performance tracking because they rate others, not themselves.

project_reviews table

- id: UUID (primary key)
- project_id: UUID (foreign key to projects.id)
- quality_rating: INTEGER (1-5)
- business_impact_rating: INTEGER (1-5)
- innovation_score: INTEGER (1-5)
- actual_roi: NUMERIC
- lessons_learned: TEXT
- reviewed_by: UUID (foreign key to users.id)
- reviewed_at: TIMESTAMP
