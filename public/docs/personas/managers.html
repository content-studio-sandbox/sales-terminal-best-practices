Manager User Guide

Overview

As a Manager, you’re responsible for creating project opportunities, reviewing applications, and building high-performing teams. This guide covers everything you need to effectively use the platform.

Your Dashboard

Manager Dashboard Features

Your dashboard provides a focused view of your projects and applications.

Key Sections

Your Projects - All projects you’ve created - Application counts per project - Project status indicators - Quick access to manage

Recent Applications - Latest applications to your projects - Candidate information - Application status - Quick review actions

Dashboard Actions

Create New Project - Click the Create Project button - Opens project creation form - Publish immediately

Review Applications - Click on any project - See all applicants - Review profiles and resumes - Accept or decline

Track Progress - Monitor application volume - See acceptance rates - Track time-to-fill

Creating Projects

Project Creation Best Practices

Creating compelling project descriptions attracts the best talent.

  • Step 1: Access Form
  • Step 2: Write Compelling Content
  • Step 3: Define Requirements
  • Step 4: Link to Ambition (Optional)
  • Step 5: Review & Publish
  1. Navigate to Leadership tab
  2. Click Projects sub-tab
  3. Click Create Project button
  4. Project form opens

Project Name - Make it exciting and clear - Include key technology or domain - Examples: - “AI Chatbot for Customer Support” - “Quantum Algorithm Development” - “Cloud Migration Automation”

Description Write a comprehensive description including:

  1. Project Overview (2-3 sentences)
    • What the project is about
    • Business problem being solved
    • Expected outcomes
  2. What You’ll Do (bullet points)
    • Specific tasks and responsibilities
    • Technologies you’ll work with
    • Team collaboration aspects
  3. What You’ll Learn (bullet points)
    • Skills to be developed
    • Technologies to master
    • Career growth opportunities
  4. Project Impact (1-2 sentences)
    • How this helps IBM
    • Real-world applications
    • Business value

Example Description:

Join our team to build an AI-powered chatbot using IBM watsonx 
Assistant. You'll help transform customer service by creating 
intelligent conversation flows and integrating with backend systems.

What You'll Do:
- Design conversation flows and intents
- Integrate watsonx Assistant with APIs
- Test and refine chatbot responses
- Collaborate with UX designers and developers

What You'll Learn:
- Natural language processing
- IBM watsonx platform
- API integration
- Agile development practices

This project will directly impact 10,000+ customers by providing 
24/7 automated support and reducing response times by 50%.

Required Skills - List 3-5 essential skills - Be realistic about requirements - Consider intern skill level - Separate with commas

Examples: - “Python, Machine Learning, Data Analysis” - “JavaScript, React, Node.js, REST APIs” - “Java, Spring Boot, SQL, Git”

Preferred Skills - Nice-to-have skills - Advanced technologies - Differentiators

Examples: - “Docker, Kubernetes, CI/CD” - “TensorFlow, PyTorch, NLP” - “AWS, Microservices, GraphQL”

Team Size - How many interns needed? - Consider project scope - Typical: 2-4 interns

Duration - Project length in months - Standard intern terms: 3-6 months - Consider onboarding time

  • Select from existing ambitions
  • Shows strategic alignment
  • Helps executives track progress
  • Not required but recommended
  1. Preview your project
  2. Check for typos
  3. Ensure all required fields complete
  4. Click Create Project
  5. Project goes live immediately!

After Publishing

What Happens Next?

  1. Project appears in intern project listings
  2. Interns can start applying immediately
  3. You receive notifications for new applications
  4. Applications appear in your dashboard

Managing Your Project

  • Edit: Update details anytime
  • Close: Stop accepting applications
  • Archive: Remove from active listings

Reviewing Applications

Application Review Workflow

When interns apply to your projects, you’ll review and make decisions.

  • Step 1: Access Applications
  • Step 2: Review Candidates
  • Step 3: Use AI Matching
  • Step 4: Make Decisions
  1. Go to your Manager Dashboard
  2. Click on a project with applications
  3. See list of all applicants
  4. Each shows:
    • Name and email
    • Application date
    • Resume (if uploaded)
    • Profile information

View Profile - Click on applicant name - See full profile - Review skills and experience - Check educational background

Download Resume - Click resume link - Opens in new tab - Review qualifications - Note relevant experience

Compare Candidates - Open multiple profiles - Compare skills side-by-side - Consider team composition - Think about diversity

For data-driven decisions, use the AI resume matcher:

  1. Navigate to Executive Dashboard
  2. Find Match Resumes to Role section
  3. Paste your project description
  4. Upload candidate resumes
  5. Get AI-powered rankings

AI Provides: - Match scores (0-100%) - Key strengths identified - Experience relevance - Skill gap analysis - Ranking recommendations

Accept Application 1. Click Accept button 2. Optionally add welcome message 3. Candidate receives notification 4. Move to next steps (onboarding)

Decline Application 1. Click Decline button 2. Optionally provide feedback 3. Be constructive and respectful 4. Candidate receives notification

Best Practices: - Respond within 3-5 business days - Provide feedback when declining - Be transparent about reasons - Keep communication professional

Selection Criteria

What to Look For:

Technical Skills - Required skills present? - Depth of experience - Relevant projects - Certifications

Soft Skills - Communication ability - Team collaboration - Problem-solving - Learning mindset

Cultural Fit - Alignment with IBM values - Enthusiasm for project - Career goals match - Growth potential

Diversity - Varied backgrounds - Different perspectives - Inclusive team building - Balanced representation

AI-Powered Resume Matching

Using the Resume Matcher

As a manager, you have access to AI-powered resume matching to make better hiring decisions.

When to Use It

  • Multiple qualified candidates
  • Need objective comparison
  • Want to reduce bias
  • Seeking data-driven insights

How It Works

  • Prepare Job Description
  • Upload Resumes
  • Review Results
  • Make Informed Decisions

Write or paste a detailed job description:

Software Engineering Intern - Cloud Platform Team

We're building scalable microservices for our cloud platform.
You'll work with Node.js, React, and Docker to create APIs
and user interfaces.

Required Skills:
- JavaScript/TypeScript
- React or Vue.js
- Node.js
- REST API design
- Git version control

Preferred Skills:
- Docker/Kubernetes
- AWS or IBM Cloud
- GraphQL
- CI/CD tools
- Agile experience

You'll learn modern cloud architecture, microservices patterns,
and DevOps practices while working on production systems.
  1. Collect candidate resumes
  2. Ensure PDF or DOCX format
  3. Upload up to 10 at once
  4. Wait for processing

AI analyzes and ranks candidates:

Match Score: Overall alignment percentage

Strengths: - Relevant skills found - Experience highlights - Education alignment

Experience: - Years of relevant work - Project experience - Internship history

Reasoning: - Why this candidate matches - Key qualifications - Standout factors

Gaps: - Missing skills - Experience shortfalls - Training needs

Use AI insights combined with: - Interview performance - Cultural fit assessment - Team needs - Growth potential

Interpreting AI Recommendations

High Match (80-100%) - Strong technical alignment - Relevant experience - Most requirements met - Priority candidate

Medium Match (60-79%) - Good foundation - Some gaps present - Training potential - Consider carefully

Low Match (0-59%) - Significant gaps - Limited relevance - May need different role - Provide feedback

Remember: AI is a tool, not the decision-maker. Use it to inform, not replace, your judgment.

Watson Orchestrate Assistant

AI Chatbot for Managers

Access the Watson Orchestrate assistant for quick information and actions.

Opening the Assistant

  1. Click chat bubble icon (bottom right)
  2. Chatbot panel opens
  3. Type your question

Useful Queries

Project Management - “Show my projects” - “How many applications do I have?” - “Which project has the most applicants?”

Candidate Information - “Find candidates with Python skills” - “Show recent applications” - “Who applied to Project X?”

Quick Actions - “Create a new project” - “Accept application from [name]” - “Send message to applicant”

Tips

  • Be specific
  • Use natural language
  • Ask follow-up questions
  • Provide context

Best Practices

Writing Great Project Descriptions

Do: - ✅ Be specific about technologies - ✅ Highlight learning opportunities - ✅ Explain business impact - ✅ Use engaging language - ✅ Include team information

Don’t: - ❌ Use jargon without explanation - ❌ Over-specify requirements - ❌ Make unrealistic promises - ❌ Copy generic descriptions - ❌ Forget to proofread

Reviewing Applications Efficiently

Set a Schedule - Review applications daily or every other day - Block time on calendar - Don’t let applications pile up

Use a Rubric - Define evaluation criteria - Score consistently - Document decisions - Compare fairly

Communicate Clearly - Respond promptly - Provide feedback - Set expectations - Keep candidates informed

Building Diverse Teams

Consider: - Different educational backgrounds - Varied skill levels - Geographic diversity - Gender balance - Underrepresented groups

Benefits: - Better problem-solving - More innovation - Broader perspectives - Stronger team dynamics

Troubleshooting

Common Issues

Can’t Create Project - Check all required fields - Ensure description is detailed enough - Verify skills are comma-separated - Try refreshing the page

Applications Not Showing - Refresh dashboard - Check project status (active?) - Verify you’re the project owner - Clear browser cache

Resume Matcher Not Working - Verify BYOK credentials configured - Test connection in Settings - Check file formats (PDF, DOCX only) - Ensure files under 10MB each

Can’t Accept/Decline Application - Check your permissions - Verify application is still pending - Try refreshing the page - Contact support if persists
