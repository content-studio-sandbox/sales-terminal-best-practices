API Reference

API Overview

Your Projects at IBM provides a RESTful API for programmatic access to platform features, including integration with Watson Orchestrate for conversational AI interactions.

Base URL

https://your-projects-np.dinero.techzone.ibm.com/api

Authentication

All API requests require authentication using a Bearer token.

curl -H "Authorization: Bearer YOUR_TOKEN" \
  https://your-projects-np.dinero.techzone.ibm.com/api/endpoint

Watson Orchestrate Integration

Your Projects integrates with Watson Orchestrate to provide conversational AI access to platform features. Users can interact with the platform using natural language through the embedded chatbot.

Setup

  1. Configure BYOK: Go to Settings → API Keys Settings
  2. Enter Watson Orchestrate credentials: API Key and Instance URL
  3. Enable auto-configuration: Automatically disables embed security
  4. Start chatting: Use the chatbot launcher (bottom-right corner)

Example Prompts

For Interns: - “What projects am I currently working on?” - “Find me AI and machine learning projects I can apply to” - “What skills should I learn to advance my career?”

For Leaders: - “Show me all pending applications for my projects” - “Who are the top 3 candidates for the Quantum Computing project?” - “Create a new project for blockchain development”

For Executives: - “Give me an overview of our talent pipeline” - “What are our top strategic ambitions?” - “Generate a quarterly talent report”

Creating Custom Skills

Watson Orchestrate skills can be created to call your-projects API endpoints. See the Watson Orchestrate Skills Guide for detailed instructions.

Key Endpoints for Skills: - GET /api/user/projects - Get user’s assigned projects - GET /api/projects/available - Find available projects - GET /api/leader/applications - Get applications for leader’s projects - GET /api/analytics/dashboard - Get dashboard metrics

REST API Endpoints

      "duration_months": 6,
      "status": "open",
      "created_at": "2025-12-01T00:00:00Z"
    }
  ],
  "total": 42,
  "limit": 50,
  "offset": 0
}

Error Responses

Standard Error Format

{
  "error": "Error message",
  "code": "ERROR_CODE",
  "details": "Additional details about the error"
}

HTTP Status Codes

  • 200 OK: Successful request
  • 201 Created: Resource created successfully
  • 400 Bad Request: Invalid request parameters
  • 401 Unauthorized: Missing or invalid authentication
  • 403 Forbidden: Insufficient permissions
  • 404 Not Found: Resource not found
  • 500 Internal Server Error: Server error

Rate Limiting

API requests are rate limited to: - 100 requests per minute per user - 1000 requests per hour per user

Rate Limit Headers:

X-RateLimit-Limit: 100
X-RateLimit-Remaining: 95
X-RateLimit-Reset: 1702857600

SDK Examples

JavaScript/Node.js

const axios = require('axios');

const api = axios.create({
  baseURL: 'https://your-projects-np.dinero.techzone.ibm.com/api',
  headers: {
    'Authorization': `Bearer ${process.env.API_TOKEN}`
  }
});

// List projects
const projects = await api.get('/projects');

// Create project
const newProject = await api.post('/projects', {
  name: 'New Project',
  description: 'Project description...',
  required_skills: ['Python', 'React'],
  team_size: 3,
  duration_months: 6
});

Python

import requests

API_BASE = 'https://your-projects-np.dinero.techzone.ibm.com/api'
headers = {'Authorization': f'Bearer {API_TOKEN}'}

# List projects
response = requests.get(f'{API_BASE}/projects', headers=headers)
projects = response.json()

# Create project
new_project = {
    'name': 'New Project',
    'description': 'Project description...',
    'required_skills': ['Python', 'React'],
    'team_size': 3,
    'duration_months': 6
}
response = requests.post(f'{API_BASE}/projects', 
                        json=new_project, 
                        headers=headers)

cURL

  https://your-projects-np.dinero.techzone.ibm.com/api/projects

Best Practices

Authentication

  • Store tokens securely
  • Rotate tokens regularly
  • Use environment variables
  • Never commit tokens to version control

Error Handling

  • Always check response status
  • Handle rate limiting gracefully
  • Implement retry logic with exponential backoff
  • Log errors for debugging

Performance

  • Use pagination for large datasets
  • Cache responses when appropriate
  • Batch requests when possible
  • Monitor API usage

Get Available Projects

GET /api/projects/available

Watson Orchestrate Configuration

Test Watson Orchestrate Connection

POST /api/test-orchestrate

Tests Watson Orchestrate API credentials.

Request Body:

{
  "apiKey": "your-service-credentials-api-key",
  "url": "https://api.us-south.watson-orchestrate.cloud.ibm.com/instances/{instance_id}/v1/..."
}

Response:

{
  "success": true,
  "message": "Successfully connected to Watson Orchestrate!"
}

Configure Watson Orchestrate Security

POST /api/configure-orchestrate-security

Automatically disables embed security for Watson Orchestrate instance.

Response:

{
  "success": true,
  "message": "Watson Orchestrate embed security has been disabled successfully. Your chatbot is now ready to use!",
  "config": {
    "is_security_enabled": false
  }
}

Get Watson Orchestrate IAM Token

POST /api/get-orchestrate-iam-token

Generates an IAM token for Watson Orchestrate authentication (used internally by the chatbot widget).

Response:

{
  "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "token_type": "Bearer"
}
