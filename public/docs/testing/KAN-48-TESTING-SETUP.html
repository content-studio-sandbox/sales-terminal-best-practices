KAN-48 Testing Framework Setup

Overview

Set up comprehensive testing infrastructure for the YourProjects application with Vitest and React Testing Library.

Testing Stack

  • Vitest: Fast unit test framework with native ESM support
  • React Testing Library: Component testing utilities
  • @testing-library/jest-dom: Custom matchers for DOM assertions
  • @testing-library/user-event: User interaction simulation
  • jsdom: DOM environment for Node.js

Known Issues

TypeScript Warnings

  • Vite version conflicts between main and vitest dependencies
  • These are warnings only and don’t affect test execution
  • Tests run successfully despite the warnings

Node Version

  • Current: v18.20.0
  • Recommended: v20+ for latest Vitest features
  • Tests work on v18 but may show engine warnings

Test Best Practices

  1. Arrange-Act-Assert Pattern

    it('should update task title', async () => {
  // Arrange
  render(<InternProjectBoard projectId="123" userId="456" />);

  // Act
  const titleInput = screen.getByRole('textbox', { name: /title/i });
  await userEvent.type(titleInput, 'New Title');

  // Assert
  expect(titleInput).toHaveValue('New Title');
});

  2. Mock External Dependencies

    • Supabase client is mocked in setup.ts
    • Add specific mocks per test as needed

  3. Test User Interactions

    • Use @testing-library/user-event for realistic interactions
    • Avoid direct DOM manipulation

  4. Accessibility Testing

    • Use semantic queries (getByRole, getByLabelText)
    • Ensures components are accessible
