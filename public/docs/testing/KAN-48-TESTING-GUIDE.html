3.4 Test Multiple Projects

To test with multiple projects:

  1. Run the test data script again (it will create another project)
  2. Or manually create projects in Supabase and assign the intern
  3. Each project should appear as a separate expandable card

  • If the intern has NO projects assigned, they should see:
    • A Monday.com-style table header (Task, Person, Status, Date, Files, Notes/Links)
    • An empty state message: ‚ÄúNo Projects Assigned Yet‚Äù
    • No ‚ÄúExplore Opportunities‚Äù button

With Projects

  • Each assigned project appears as an expandable card
  • Clicking a project card expands/collapses the task board
  • The task board shows all tasks for that specific project
  • All CRUD operations work (Create, Read, Update, Delete)
  • Status changes are reflected immediately
  • Comments and file attachments work

Troubleshooting

‚ÄúNo Projects Assigned Yet‚Äù shows even after running test data

  1. Check if the migration was applied:

    SELECT * FROM intern_project_tasks LIMIT 1;

    If this returns an error, the migration wasn‚Äôt applied.

  2. Check if the intern is assigned to the project:

    SELECT ps.*, p.name as project_name
FROM project_staff ps
JOIN projects p ON p.id = ps.project_id
JOIN users u ON u.id = ps.user_id
WHERE u.email = 'intern.local@ibm.com';

  3. Check if tasks exist:

    SELECT t.*, p.name as project_name
FROM intern_project_tasks t
JOIN projects p ON p.id = t.project_id
JOIN project_staff ps ON ps.project_id = p.id
JOIN users u ON u.id = ps.user_id
WHERE u.email = 'intern.local@ibm.com';

Tasks not showing in the board

  1. Make sure you‚Äôve expanded the project card (click on it)
  2. Check browser console for errors
  3. Verify the project_id matches between project_staff and intern_project_tasks

File uploads not working

  1. Check if the project-files storage bucket exists in Supabase
  2. Verify RLS policies allow the intern to upload files
  3. Check browser console for storage errors
