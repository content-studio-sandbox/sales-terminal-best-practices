Platform Architecture

Architecture Components

Frontend Layer

  • React + Carbon Design: Modern, accessible UI built with IBM‚Äôs Carbon Design System
  • TypeScript: Type-safe development with enhanced IDE support
  • Vite: Lightning-fast build tool and dev server
  • React Router: Client-side routing for seamless navigation
  • Supabase Client: Real-time data synchronization and authentication

API Server

  • Node.js + Express: RESTful API handling business logic
  • PDF/DOCX Parsing: Extract text from resume uploads using pdf-parse and mammoth
  • API Key Management: Secure BYOK credential storage with encryption
  • Test Connection Endpoints: Validate Watson service configurations
  • Static File Serving: Hosts React app and Quarto documentation

Database Layer

  • Supabase (PostgreSQL): Managed database with real-time capabilities
  • Row Level Security (RLS): Fine-grained access control at database level
  • Service Role Key: Admin access for server-side operations
  • Real-time Subscriptions: Live updates for collaborative features
  • Automated Backups: Point-in-time recovery and data protection

Authentication

  • OAuth2 Proxy: Enterprise SSO integration (SAML, OIDC)
  • Session Management: Secure cookie-based authentication
  • Role-Based Access: Executive, Manager, and Intern permissions
  • API Key Encryption: AES-256 encryption for stored credentials

IBM watsonx.ai

  • Resume Analysis: Extract skills, experience, and qualifications
  • Semantic Matching: Rank candidates against job requirements
  • Granite Models: IBM‚Äôs enterprise-grade foundation models
  • Detailed Reasoning: Explainable AI with match justifications
  • BYOK Support: Use your own IBM Cloud API keys

Watson Orchestrate

  • Conversational AI: Natural language interface for queries
  • Workflow Automation: Streamline repetitive tasks
  • Skill Integration: Connect to enterprise systems and APIs
  • Context Awareness: Maintain conversation history and user preferences
  • Test Connection: Validate configuration before deployment

Data Flow

  1. User Authentication: OAuth2 Proxy validates SSO credentials and creates session
  2. Resume Upload: Frontend sends PDF/DOCX to API server for text extraction
  3. AI Analysis: API calls watsonx.ai with resume text and job requirements
  4. Ranking Results: AI returns scored candidates with detailed reasoning
  5. Database Storage: Results cached in Supabase for quick retrieval
  6. Real-time Updates: Supabase broadcasts changes to connected clients
  7. Orchestrate Queries: Users interact with Watson Orchestrate for insights and automation

Security & Compliance

  • Encryption at Rest: All sensitive data encrypted in database
  • Encryption in Transit: TLS 1.3 for all API communications
  • API Key Isolation: Each user‚Äôs credentials stored separately with encryption
  • Audit Logging: Track all API calls and data access
  • GDPR Compliance: Data retention policies and user data export
